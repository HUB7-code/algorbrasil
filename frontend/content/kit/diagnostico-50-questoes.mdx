Este diagnóstico revela com precisão onde sua organização está na jornada de Governança de IA. Responda com honestidade — cada lacuna é uma vulnerabilidade que o PL 2338 pode transformar em passivo jurídico.

> **Resultado:** Some os pontos ao final de cada dimensão e consulte a tabela de maturidade.

## DIMENSÃO 1 — ESTRATÉGIA (Questões 1–12)

*Avalia se a liderança tem visão, governança e política definidas para IA*

| # | Questão | 0 | 1 | 2 |
|---|---------|---|---|---|
| 1 | A organização possui uma **Política de IA** formalmente aprovada pela alta direção? | | | |
| 2 | Existe um **Comitê ou responsável formal** (AI Officer, Comitê de IA) com mandato definido? | | | |
| 3 | A liderança (C-Level) recebe **treinamento mínimo anual** sobre riscos de IA? | | | |
| 4 | O plano estratégico da organização menciona **metas de uso responsável de IA**? | | | |
| 5 | A organização possui **orçamento específico** destinado à Governança de IA? | | | |
| 6 | Existe um processo de **aprovação formal** antes da adoção de qualquer nova ferramenta de IA? | | | |
| 7 | A organização já realizou um **mapeamento de Shadow AI** (IA não-oficial em uso)? | | | |
| 8 | Existem **KPIs de governança de IA** acompanhados periodicamente pela diretoria? | | | |
| 9 | A organização possui um **Plano de Resposta a Incidentes de IA** (falhas, viés, vazamentos)? | | | |
| 10 | Os **contratos de parceiros e fornecedores** incluem cláusulas de conformidade em IA? | | | |
| 11 | A política de IA está **integrada com a política de privacidade (LGPD)** existente? | | | |
| 12 | Há uma **estratégia de comunicação** para informar colaboradores e clientes sobre o uso de IA? | | | |

## DIMENSÃO 2 — ESTRUTURA (Questões 13–24)

*Avalia se papéis, responsabilidades e controles estão formalizados*

| # | Questão | 0 | 1 | 2 |
|---|---------|---|---|---|
| 13 | Existe um **inventário atualizado** de todos os sistemas de IA em uso na organização? | | | |
| 14 | Cada sistema de IA tem um **responsável (Gestor de IA) identificado por CPF/matrícula**? | | | |
| 15 | Os sistemas de IA estão **classificados por nível de risco** (Baixo, Alto, Excessivo) conforme PL 2338? | | | |
| 16 | Existe uma **Avaliação de Impacto Algorítmico (AIA)** para os sistemas de Risco Alto? | | | |
| 17 | Os sistemas de Risco Alto possuem **mecanismo de Supervisão Humana ativado e documentado**? | | | |
| 18 | Há um processo formal para **onboarding de fornecedores de IA** com due diligence documentada? | | | |
| 19 | A organização mantém um **registro de logs de decisões automatizadas** por período mínimo de 1 ano? | | | |
| 20 | Existem **controles de acesso** adequados para sistemas de IA que tratam dados sensíveis? | | | |
| 21 | A organização possui **contrato de processamento de dados (DPA)** com todos os fornecedores de IA? | | | |
| 22 | Há um processo de **revisão periódica (auditoria interna)** dos sistemas de IA em produção? | | | |
| 23 | A organização possui **política de gestão de modelos** (versionamento, retreinamento, descontinuação)? | | | |
| 24 | Existe um **canal de reporte** para colaboradores que identifiquem riscos ou anomalias em sistemas de IA? | | | |

| 26 | Os sistemas de IA passam por **testes de performance e viés** antes de qualquer atualização de modelo? | | | |
| 27 | Há um processo de **aprovação em estágios (staging → homologação → produção)** para sistemas de IA? | | | |
| 28 | Existe um **SLA definido** para resposta a incidentes causados por falhas de sistemas de IA? | | | |
| 29 | A organização consegue **explicar as decisões** de seus sistemas de IA a um titular de dados ou regulador? | | | |
| 30 | Existe processo formal para **receber e tratar reclamações** de pessoas afetadas por decisões automatizadas? | | | |
| 31 | Os processos de IA são **documentados em linguagem acessível** (não apenas técnica)? | | | |
| 32 | Há monitoramento contínuo de **drift de modelo** (degradação de performance ao longo do tempo)? | | | |
| 33 | A organização possui um **plano de contingência** para operação sem os sistemas de IA críticos? | | | |
| 34 | Existem processos documentados para garantir a **qualidade e limpeza dos dados** de entrada? | | | |
| 35 | A organização realiza **testes adversariais** (tentativas de enganar/manipular o modelo)? | | | |
| 36 | Há um processo para **descontinuação segura** de sistemas de IA (descomissionamento de dados e modelos)? | | | |
| 37 | Os processos de IA estão documentados de forma que possam ser **auditados externamente sem preparação prévia**? | | | |

| # | Questão | 0 | 1 | 2 |
|---|---------|---|---|---|
| 38 | Os colaboradores que operam sistemas de IA recebem **treinamento específico** sobre o sistema e seus limites? | | | |
| 39 | A organização tem um **programa de conscientização sobre riscos de IA** para todos os funcionários? | | | |
| 40 | Os gestores sabem identificar o que é **Shadow AI** e como reportá-la? | | | |
| 41 | Existe uma **cultura de questionamento** onde colaboradores podem contestar decisões de sistemas de IA? | | | |
| 42 | A organização mede o **nível de confiança e adoção responsável de IA** pelos colaboradores? | | | |
| 43 | Há comunicação regular da **liderança sobre uso ético e responsável de IA** para toda a organização? | | | |
| 44 | Existem **políticas claras** sobre o que colaboradores podem e não podem usar IA para fazer? | | | |
| 45 | A organização possui um **código de ética de IA** ou integrou princípios de ética ao código de conduta existente? | | | |
| 46 | Há **diversidade de perspectivas** nas equipes que desenvolvem/supervisionam sistemas de IA (gênero, etnia, área)? | | | |
| 47 | A organização tem parceria com **instituições acadêmicas ou associações** para atualização sobre IA ética? | | | |
| 48 | Colaboradores sabem como proceder em caso de **erro ou comportamento inesperado** de um sistema de IA? | | | |
| 49 | A organização celebra e comunica **casos de uso responsável de IA** internamente (casos de sucesso)? | | | |
